# 사이트 초기화와 더미 데이터 기준

이 문서는 로컬 또는 스테이징 QA 환경에서 관리자 계정을 보존한 채 사이트 운영 데이터를 다시 채우는 기준을 정리한다. 운영 DB에서 실행하는 절차가 아니다.

## 초기화 범위

사이트 초기화는 설치 자체를 되돌리는 작업이 아니라, 검수용 도메인 데이터를 비우고 실제 등록 경로로 다시 만드는 작업이다.

보존한다.

- 설치 잠금 파일과 `config/config.php`
- `sr_modules`의 설치/활성 상태와 적용된 schema version
- 최초 owner 또는 테스트 관리자 계정
- 관리자 권한, 메뉴 override, 업데이트 이력처럼 관리자 진입에 필요한 운영 기반
- 보안/개인정보 보관 정책 설정

초기화 대상이다.

- 관리자 계정 외 테스트 회원 계정과 회원 프로필/닉네임/세션/동의/인증 로그
- 회원 그룹, 자동 배정 규칙, 수동 배정, 평가 로그
- 콘텐츠 그룹, 콘텐츠, 콘텐츠 시리즈, 콘텐츠 댓글, 콘텐츠 파일과 관련 접근/다운로드/자산 로그
- 커뮤니티 게시판 그룹, 게시판, 카테고리, 글, 댓글, 신고, 스크랩, 쪽지, 시리즈, 첨부와 관련 자산 로그
- 배너, 팝업레이어, 사이트 메뉴 항목, SEO 운영 설정처럼 표시 검수용으로 만든 사이트 운영 데이터
- 쿠폰 정의, 지급, 사용, 환불 로그
- 포인트/적립금/예치금 테스트 조정 거래, 환전 로그, 출금/환불 신청
- 알림과 발송/읽음 상태
- 감사 로그 중 더미 데이터 등록으로 생긴 로그. 다만 추적을 위해 보존할 필요가 있으면 초기화 기록으로 분리한다.

초기화 전에는 현재 DB가 로컬 또는 스테이징인지 확인한다. `config/config.php`가 운영 도메인, 운영 DB 이름, 운영 storage를 가리키면 실행하지 않는다.

`.tools/bin/seed-dummy-http.php`를 사용할 때는 다음처럼 mutation 허용 플래그를 명시한다. 이 스크립트는 회원, 콘텐츠, 커뮤니티, 배너, 팝업레이어, 쿠폰, 알림 데이터를 실제 HTTP 경로로 만들고 일부 표시 데이터를 정리할 수 있으므로 기본 실행은 거부한다.

```sh
SR_SEED_ALLOW_MUTATION=1 \
SR_SEED_BASE_URL=http://127.0.0.1:8080 \
SR_SEED_ADMIN_IDENTIFIER=admin \
SR_SEED_ADMIN_PASSWORD='password' \
php .tools/bin/seed-dummy-http.php
```

시더는 관리자 오류, 공개 피드백 오류, 회원가입 오류 목록이 POST 응답에 남으면 즉시 실패해야 한다. 응답이 200/302처럼 정상으로 보여도 도메인별 생성 전/후 카운트가 기대 증가량보다 작으면 더미 등록 실패로 기록한다.

## 관리자 계정 기준

더미 데이터 등록에 사용할 관리자 계정은 최소 하나만 필요하다.

- 관리자 계정은 삭제하거나 재생성하지 않는다.
- 관리자 비밀번호는 테스트 환경에서만 사용하고 릴리스 문서나 공개 기록에 남기지 않는다.
- 관리자 계정의 표시명, 이메일, 권한은 더미 데이터 변동과 독립적으로 유지한다.
- 테스트 회원을 관리자로 승격해 쓰는 경우 초기화 대상에서 제외할 계정을 명시한다.

## 등록 경로 기준

더미 데이터는 DB insert로 만들지 않고 실제 등록/저장 경로를 거친다.

- 회원은 `/admin/members/save` 또는 `/register`를 사용한다.
- 회원 그룹은 `/admin/member-groups` 또는 `/admin/member-groups/save`를 사용한다.
- 콘텐츠 그룹은 `/admin/content-groups`, 콘텐츠는 `/admin/content/save`를 사용한다.
- 커뮤니티 게시판 그룹과 게시판은 `/admin/community/board-groups`, `/admin/community/boards/create`를 사용한다.
- 커뮤니티 글/댓글/쪽지/신고는 회원 로그인 후 공개 커뮤니티 경로를 사용한다.
- 배너와 팝업레이어는 `/admin/banners/save`, `/admin/popup-layers/save`를 사용한다.
- 쿠폰은 `/admin/coupons`의 정의 생성과 지급 action을 사용한다.
- 알림은 `/admin/notifications/create` 또는 `/admin/notifications`의 생성 action을 사용한다.

시더나 수동 등록 스크립트는 각 POST 전에 GET 화면에서 CSRF 토큰을 받아야 한다. POST 이후에는 목록 카운트나 상세 화면 조회로 실제 생성 여부를 확인한다.

## 더미 데이터 수량

전체 화면 검수용 기본 수량은 도메인별 10-15건이다. 한 화면이 페이지네이션, 필터, 상태 뱃지, 정렬, 공개/비공개 분기를 모두 보여야 하면 15건을 우선한다.

권장 기본 세트:

- 테스트 회원 12명
- 회원 그룹 12개
- 콘텐츠 그룹 12개와 콘텐츠 12개
- 커뮤니티 게시판 그룹 12개, 게시판 12개, 게시글 12개
- 배너 12개와 팝업레이어 12개
- 쿠폰 정의 12개
- 사이트 알림 12개

커뮤니티 게시글 제목과 본문은 `.tools/fixtures/community-literature-texts.php`에 보관한 공개 영역 근대 조선 문학 작품 원문 전문과 더미 생성 문구를 섞어 사용한다. 목록, 상세, 검색 화면에서 긴 한국어 본문, 제목 길이, 문장 길이, 줄바꿈을 검수하기 위한 값이며, 운영 콘텐츠 원문으로 보존할 목적은 아니다.

자산 거래, 환전, 출금/환불 신청처럼 돈성 상태가 생기는 데이터는 별도 테스트 목적이 있을 때만 만든다. 금액성 더미 데이터는 환불/회수/정산 상태를 검증할 필요가 있는 QA 기록에 실행 목적과 cleanup 기준을 함께 남긴다.

## 완료 기록

더미 등록 작업을 완료하면 다음을 남긴다.

- 실행한 base URL
- 사용한 등록 경로 목록
- 도메인별 생성 전/후 카운트
- 실패한 경로와 실패 사유
- `php .tools/bin/check.php` 결과
- 가능한 경우 HTTP smoke 결과

이 문서 기준을 바꾸는 코드 변경이 있으면 저장소 문서를 같은 작업에서 갱신한다. GitHub Wiki는 현재 운영 문서의 정본으로 사용하지 않는다.
