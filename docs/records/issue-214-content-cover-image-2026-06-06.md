# Issue 214 Content Cover Image

## 결정

- 콘텐츠 대표 이미지는 콘텐츠 모듈이 `sr_content_items.cover_image_url`로 소유한다.
- 관리자 콘텐츠 생성/수정 화면은 URL 입력과 이미지 파일 업로드를 모두 제공한다.
- 업로드 파일은 jpg, png, webp만 허용하고 재인코딩 후 `content/cover-images/{Y}/{m}` 저장소 키로 저장한다.
- 업로드 커버는 저장소 종류와 관계없이 `/content/cover-image?file=...` 프록시 URL로 저장하고 출력한다.
- 관리자 콘텐츠 수정 화면은 현재 커버 미리보기와 삭제 체크박스를 제공한다.
- 업로드 커버가 교체되거나 삭제되고 다른 콘텐츠가 같은 프록시 URL을 참조하지 않으면 저장소 파일을 정리한다.
- 콘텐츠 저장 후 시리즈나 파일 연결 같은 후속 저장이 실패해도 새 커버가 이미 콘텐츠에 연결된 경우 이전 커버 저장소 정리는 시도한다.
- 콘텐츠 저장 후 후속 저장이 실패한 부분 성공 상태는 별도 감사 로그에 커버 이미지 변경과 저장소 정리 결과를 남긴다.
- 당시 정책: 콘텐츠 그룹 삭제 시 그룹 콘텐츠의 커버 이미지 프록시 URL도 같은 기준으로 저장소에서 정리하며, 실패하면 저장소 정리 실패 기록에 남긴다.
- 당시 정책: 콘텐츠 그룹 삭제 후 커버 저장소 정리 중 예외가 나도 그룹 삭제 완료 흐름을 막지 않고, 가능한 경우 저장소 정리 실패 기록과 로그로 남긴다.
- 2026-06-14 현재 정책: 콘텐츠 그룹 삭제는 콘텐츠를 삭제하지 않고 그룹 연결만 해제하므로 커버 이미지 저장소 파일 정리를 실행하지 않는다. 커버 정리는 콘텐츠 삭제, 커버 교체, 현재 이미지 삭제 흐름에서 처리한다.
- 콘텐츠 저장 감사 로그에는 커버 이미지 변경, 삭제, 업로드, 저장소 정리 결과를 메타데이터로 남긴다.
- 커버 이미지가 없으면 공개 홈과 그룹 목록에서 회색 플레이스홀더를 출력한다.

## 반영 범위

- 콘텐츠 홈 최신 1건 히어로와 최근 콘텐츠 목록에서 커버 이미지를 사용한다.
- 콘텐츠 그룹 목록 썸네일과 콘텐츠 상세 상단 커버에 같은 값을 사용한다.
- 콘텐츠 상세 OG 이미지에도 커버 이미지 URL을 연결한다.
- 콘텐츠 복사는 원본 커버 이미지 URL을 그대로 복사한다. 같은 저장소 프록시 URL을 여러 콘텐츠가 함께 쓰면 한 콘텐츠의 교체/삭제가 공유 파일을 제거하지 않는다.
