# 마일스톤 32 커뮤니티 홈 피드 의미 결정

작성일: 2026-06-24

관련 이슈: #359

## 결정

커뮤니티 홈의 최신글과 인기글은 전역 top-N 의미로 정한다.

- 최신글: 읽기 가능한 전체 게시판에서 최신글 10개
- 인기글: 읽기 가능한 전체 게시판에서 조회수 기준 인기글 5개

기존 구현은 읽기 가능한 게시판을 순회하면서 게시판별 최신글 5개와 인기글 5개를 가져온 뒤 PHP에서 다시 정렬했다. 이 방식은 게시판 다양성을 보장하는 효과가 있지만, 게시판 수가 늘면 홈 요청마다 최소 `2N`개의 게시글 목록 조회가 발생한다.

이번 결정은 홈의 의미를 전체 커뮤니티의 최신/인기 흐름으로 명확히 하고, 최신글과 인기글을 각각 하나의 통합 쿼리로 가져오도록 바꾼다.

## 유지하는 제한

- 읽기 가능한 게시판만 후보에 포함한다.
- `p.status = 'published'` 글만 후보에 포함한다.
- secret post 본문 excerpt 제한을 유지한다.
- 유료 읽기 게시판은 home excerpt와 이미지 노출 제한을 유지한다.
- 대표 이미지는 active image attachment 중 가장 작은 id를 선택하는 기존 목록 의미를 유지한다.
- 인기글 reaction count는 최종 인기글 5개에 대해서만 조회한다.

## 의도적으로 바뀌는 점

- 큰 게시판 하나가 최신글 또는 인기글 슬롯을 더 많이 차지할 수 있다.
- 게시판별 5개 선별 후 전역 정렬하던 균형 샘플 의미는 더 이상 홈 피드의 기준이 아니다.
- 게시판별 summary/cache substrate는 사용하지 않으므로 #360의 board summary 계약에 의존하지 않는다.

## 성능 효과

- 최신글 조회는 게시판별 `N`개 쿼리에서 통합 1개 쿼리로 줄어든다.
- 인기글 조회도 게시판별 `N`개 쿼리에서 통합 1개 쿼리로 줄어든다.
- 최신 댓글과 시리즈 조회는 이미 읽기 가능한 board id 집합을 이용한 통합 쿼리 형태라 이번 범위에서는 유지한다.

## 후속 관찰

- view_count 지연 병합이 #362에서 도입되면 인기글 정렬의 staleness 허용 범위를 다시 확인한다.
- 특정 게시판 독점이 운영상 문제가 되면 별도 이슈에서 게시판 균형 샘플 또는 board summary/cache substrate를 #360 계약에 맞춰 재검토한다.

## 검증

구현 변경은 `php .tools/bin/check.php`와 HTTP smoke로 확인한다. 문서 변경은 `git diff --check`를 함께 확인한다.
