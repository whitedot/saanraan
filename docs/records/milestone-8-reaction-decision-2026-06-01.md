# 마일스톤 8 리액션 범위 결정

## 결정

- 마일스톤 8에서는 커뮤니티 게시글 리액션 테이블, 공개 버튼, 관리자 집계 화면을 새로 추가하지 않는다.
- 현재 반응 기능은 커뮤니티 스크랩과 콘텐츠 완료 버튼으로 유지한다.
- 별도 리액션을 도입하려면 계정별 중복 집계 기준, 익명/탈퇴 회원 처리, 개인정보 보존 기간, 신고·운영 정책이 먼저 정해져야 한다.

## 이유

- 커뮤니티 스크랩은 이미 개인 보관과 개인정보 사본 제공 범위를 갖고 있다.
- 콘텐츠 완료 버튼은 콘텐츠 모듈의 자산/활동 정책과 연결되어 있어 리액션과 다른 도메인 의미를 갖는다.
- 단순 좋아요를 지금 추가하면 스크랩, 완료 버튼, 향후 추천/랭킹 정책이 섞일 수 있어 마일스톤 8의 운영 정합성 범위를 넘는다.

## 후속 조건

- 새 마일스톤에서 리액션을 다룰 때는 모듈 소유 테이블, privacy export/cleanup, 관리자 표시 범위, 스팸/오용 제한을 함께 설계한다.
