# #209 게시판·콘텐츠 운영 단위 삭제

## 반영 범위

- 커뮤니티 게시판, 커뮤니티 게시판 그룹, 콘텐츠 그룹, 콘텐츠 시리즈의 관리자 삭제 POST 흐름을 추가했다.
- 삭제 요청은 CSRF 검증과 관리자 `delete` 권한을 요구한다.
- 삭제 가능 여부와 실제 삭제는 각 도메인 모듈 helper가 판단하며, core에는 도메인 삭제 정책을 추가하지 않았다.
- 삭제 성공 시 `sr_audit_log()`에 대상 key/title과 삭제·참조 개수를 기록한다.

## 삭제 정책

- 커뮤니티 게시판은 게시글이 있어도 삭제할 수 있다. 삭제 성공 시 게시판 설정, 설정 소스, 카테고리, 게시글, 댓글, 스크랩, 링크 참조, 커뮤니티 시리즈/항목, 첨부 DB 행과 저장소 파일을 함께 삭제한다.
- 커뮤니티 게시판 그룹은 연결 게시판이 있으면 삭제하지 않는다. 삭제 성공 시 그룹 설정을 함께 삭제한다.
- 콘텐츠 그룹은 연결 콘텐츠가 있어도 삭제할 수 있다. 삭제 성공 시 그룹 설정, 콘텐츠, 댓글, 링크 참조, 리비전, 콘텐츠 시리즈 항목 연결, 파일 링크, 파일 DB 행과 저장소 파일을 함께 삭제한다.
- 콘텐츠 시리즈는 콘텐츠 자체를 삭제하지 않고 `sr_content_series_items` 연결을 삭제한 뒤 시리즈를 삭제한다.
- 사이트 메뉴, 배너/팝업 exact target, 쿠폰 target, 링크 카드, 콘텐츠 그룹 초기화면 참조는 가능한 범위에서 외부 운영 참조로 확인하며, 참조가 있으면 삭제를 차단한다. 해당 모듈이나 테이블이 없으면 0건으로 처리한다.
- 포인트, 적립금, 예치금, 쿠폰 접근권 사용처럼 금전성/혜택성 처리 흔적을 담는 자산 로그와 파일 다운로드 로그는 삭제하지 않는다. 삭제 대상 콘텐츠나 게시글에 대한 현재 접근권/권한 상태 데이터는 대상이 사라지면 효력이 없으므로 함께 정리한다.

## 검증

- `php .tools/bin/check.php`
- 개별 변경 파일 `php -l`
