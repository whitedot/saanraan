# 관리자 목록 컬럼 기준

관리자 목록은 운영자가 행을 식별하고 상태를 판단한 뒤 바로 조치할 수 있게 구성한다. 데스크톱에서는 현재 화면의 전체 업무 맥락을 보여주고, 모바일이나 좁은 화면에서는 식별, 상태, 관리 액션에 필요한 열을 우선 남긴다.

## 공통 기준

- 식별 열: 제목, 이름, 회원 공개 이름/이메일, key, slug처럼 운영자가 대상을 찾는 값이다.
- 상태 열: 공개/비공개, 회원 상태, 처리 상태, 발송 상태처럼 행의 현재 판단에 필요한 값이다.
- 액션 열: 보기, 수정, 삭제, 환불, 조정, 처리 변경처럼 행 단위 작업을 수행하는 값이다.
- 보조 열: 생성일, 수정일, 공개일, 정렬, 수량, 스킨, 노출 위치, 참조, 클릭 수처럼 판단을 돕지만 작은 화면에서 숨겨도 되는 값이다.
- 내부 DB ID는 목록 본문에 별도 열로 노출하지 않는다. 작업에 필요한 ID는 hidden input, URL 파라미터, `data-*` 속성으로만 사용한다.
- 회원 식별은 회원 목록 기준을 따른다. 일반 관리자 목록에서는 회원 공개 해시보다 `sr_admin_member_display_name_preview()`와 `sr_admin_member_email_display()`를 우선 표시한다. 닉네임 사용 시 공개 이름은 닉네임을 우선 사용하며, 회원 목록은 운영자가 원문 닉네임을 바로 대조할 수 있도록 공개 이름 옆에 닉네임 컬럼을 함께 표시한다.
- 헤더 정렬을 제공하는 목록은 서버 allowlist에 있는 컬럼만 정렬 파라미터로 받는다. 헤더의 작은 오름/내림 버튼으로 정렬 기준과 방향을 선택하고, `sort`/`dir` query string은 검색/필터와 함께 유지한다. 기본 정렬 초기화는 건수 표시 앞의 아이콘 버튼으로 제공하며 정렬 파라미터와 page만 제거한다. 한 화면에 독립 목록이 둘 이상이면 `asset_sort`/`asset_dir`, `assignment_sort`/`assignment_dir`처럼 목록별 query key를 분리한다.
- 위계 편집이나 순서 저장 자체가 핵심인 메뉴 트리형 목록은 헤더 정렬을 적용하지 않는다.
- 선택 기반 일괄 작업을 제공하는 목록은 선택 열을 첫 열로 두고 핵심 컬럼으로 취급한다. 선택 열은 현재 페이지 선택을 뜻하며, 체크박스만 담는 `admin-table-checkbox-cell` 고정폭 셀로 표시한다. 검색 결과 전체 선택은 별도 안내와 서버 재계산 계약이 있는 작업에서만 추가한다.
- 일괄 작업 후보는 위험도와 실행 모델을 함께 정리한다. 단순 상태 변경은 즉시 제한형 후보가 될 수 있지만, 회원 그룹 재평가, 대량 재계산, 복사, 발송, 금전성 승인/반려처럼 고부하 가능 또는 고위험 작업은 작업 테이블형이나 고위험 즉시 제한형 carve-out 조건을 먼저 검토한다.

## 변경 절차

목록 컬럼을 추가하거나 제거할 때는 다음 항목을 같이 확인한다.

- view의 `thead`, `tbody`, 빈 상태 `colspan`
- 목록별 CSS의 폭 지정과 좁은 화면 숨김 규칙
- 이 문서와 Wiki의 관리자 화면별 항목 설명
- 회원 또는 자산 목록이라면 공개 해시, 내부 ID, 회원 표시 기준이 공통 기준을 벗어나지 않는지 여부
