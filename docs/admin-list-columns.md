# 관리자 목록 컬럼 기준

관리자 목록은 운영자가 행을 식별하고 상태를 판단한 뒤 바로 조치할 수 있게 구성한다. 데스크톱에서는 현재 화면의 전체 업무 맥락을 보여주고, 모바일이나 좁은 화면에서는 식별, 상태, 관리 액션에 필요한 열을 우선 남긴다.

## 공통 기준

- 식별 열: 제목, 이름, 회원 표시명/이메일, key, slug처럼 운영자가 대상을 찾는 값이다.
- 상태 열: 공개/비공개, 회원 상태, 처리 상태, 발송 상태처럼 행의 현재 판단에 필요한 값이다.
- 액션 열: 보기, 수정, 삭제, 환불, 조정, 처리 변경처럼 행 단위 작업을 수행하는 값이다.
- 보조 열: 생성일, 수정일, 공개일, 정렬, 수량, 스킨, 출력 위치, 참조, 클릭 수처럼 판단을 돕지만 작은 화면에서 숨겨도 되는 값이다.
- 내부 DB ID는 목록 본문에 별도 열로 노출하지 않는다. 작업에 필요한 ID는 hidden input, URL 파라미터, `data-*` 속성으로만 사용한다.
- 회원 식별은 회원 목록 기준을 따른다. 일반 관리자 목록에서는 회원 공개 해시보다 `sr_admin_member_display_name_preview()`와 `sr_admin_member_email_display()`를 우선 표시한다.

## 변경 절차

목록 컬럼을 추가하거나 제거할 때는 다음 항목을 같이 확인한다.

- view의 `thead`, `tbody`, 빈 상태 `colspan`
- 목록별 CSS의 폭 지정과 좁은 화면 숨김 규칙
- 이 문서와 Wiki의 관리자 화면별 항목 설명
- 회원 또는 자산 목록이라면 공개 해시, 내부 ID, 회원 표시 기준이 공통 기준을 벗어나지 않는지 여부
