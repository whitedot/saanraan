# 산란 특장점 소개

이 문서는 `saanraan`의 현재 구현 기능을 제품 소개나 기능 안내에 활용할 수 있도록 정리한 목록이다. 실제 저장소에 포함된 본체와 번들 모듈을 기준으로 작성하며, 본인확인, 결제, 회원 마이그레이션처럼 계획 단계에 있는 기능은 구현 기능으로 설명하지 않는다.

`saanraan`은 저가형 PHP 웹호스팅에서도 올려 쓸 수 있는 회원 중심 모듈형 웹 솔루션 베이스다. 코어는 설치, 요청 진입, 설정, 보안 helper처럼 실행 기반에 집중하고, 회원·콘텐츠·커뮤니티·자산·쿠폰·알림·개인정보·사이트 구성은 모듈이 각자 책임져 필요한 기능부터 차례로 확장할 수 있다.

## 한눈에 보는 특장점

| 기능 | 안내 문구 | 주요 기능 |
| --- | --- | --- |
| 모듈형 구성 | 필요한 기능만 골라 설치하고 켤 수 있어, 작은 회원 사이트에서 시작해 콘텐츠·커뮤니티·자산 기능을 단계적으로 붙이기 쉽다. | 회원, 관리자, 개인정보, 콘텐츠, 커뮤니티, 사이트 운영, 자산 모듈 번들 제공 |
| 공유호스팅 친화 구조 | 별도 애플리케이션 서버나 복잡한 빌드 파이프라인 없이 PHP 파일과 DB 중심으로 배포할 수 있어 서버 선택의 부담을 낮춘다. | PHP 8.1 이상, MySQL 또는 MySQL 호환 DB, Vanilla JavaScript, plain CSS |
| 회원 중심 확장 | 계정, 그룹, 자산, 쿠폰, 알림, 개인정보 요청이 같은 회원 식별자를 기준으로 이어져 기능을 따로 붙여도 흐름이 흩어지지 않는다. | 회원 관리, 회원 그룹, 포인트/적립금/예치금, 쿠폰·이용권, 알림, 개인정보 요청 |
| 콘텐츠·커뮤니티 연결 | 콘텐츠와 커뮤니티가 서로를 복사하거나 소유하지 않고 일반 링크/인용 HTML 또는 텍스트 링크로 이어질 수 있어 안내 글, 후기, 공지, 토론을 자연스럽게 연결한다. | 모듈 간 대상 검색 삽입, 콘텐츠↔커뮤니티 공개 링크, 토큰 저장 거부 |
| 운영 안전성 | 권한 확인, CSRF 보호, 감사 로그, 보존 정책을 기본 흐름에 넣어 중요한 변경이 언제 누구에 의해 일어났는지 되짚기 쉽다. | 관리자 권한, 감사 로그, 보존 정책, CSRF, 출력 escape, rate limit |
| 작은 코어 | 코어가 도메인 욕심을 내지 않아 콘텐츠, 커뮤니티, 쿠폰, 자산 정책을 각 모듈 안에서 읽고 고칠 수 있다. | 설치/업데이트, DB 연결, 설정, 모듈 조회, 보안 helper, 출력 슬롯 |

## 설치와 기본 관리

| 기능 | 안내 문구 | 주요 기능 |
| --- | --- | --- |
| 웹 설치 | 브라우저에서 설치 흐름에 들어갈 수 있어 SSH나 CLI 접근이 어려운 호스팅에서도 초기 구성을 시작하기 쉽다. | 미설치 상태 자동 설치 흐름, DB 기반 사이트 설정 저장 |
| 모듈 설치와 활성화 | 기능을 모듈 단위로 설치·활성화하므로 당장 쓰지 않는 기능을 숨기고 필요한 때에만 열 수 있다. | 모듈 설치, 활성화, 비활성화, 모듈 상태 관리 |
| 업데이트 관리 | 본체와 모듈의 schema version을 따로 확인해 어떤 SQL이 적용됐고 무엇이 남았는지 화면에서 판단할 수 있다. | 본체 schema version, 모듈 schema version, update SQL 구조 |
| 배포 친화 구성 | 파일 배치와 DB 업데이트가 분리되어 FTP/SFTP, Git checkout, zip 배포 같은 여러 방식에 맞춰 반영할 수 있다. | PHP 파일 기반 배포, Apache 호환 보호 기준, `sr_` DB prefix |
| 저장소 기반 | 로컬 파일과 S3 호환 저장소를 공통 helper로 다룰 수 있어 업로드 기능이 모듈마다 제각각 흩어지는 일을 줄인다. | 로컬 파일 저장 helper, S3 호환 저장소 helper |

## 관리자 대시보드와 관리 도구

| 관리자 메뉴 | 모듈이 제공하는 기본 메뉴 위에 표시 순서, 숨김 설정, 모듈 그룹 아이콘 설정을 얹어 실제 업무 동선에 맞는 메뉴 구조를 만들 수 있다. | 관리자 메뉴 관리, 관리자 화면 이동 구조 |

| 보존 정책 | 로그와 요청 데이터를 언제까지 둘지 별도 기준으로 관리해 장기 보관 부담과 확인 필요성을 함께 조절할 수 있다. | 운영 데이터 보존 정책 관리 |
| 관리자 UI | 목록, 필터, 페이지네이션, 모달, 도움말 같은 공통 패턴을 맞춰 여러 모듈 화면을 익숙한 방식으로 사용할 수 있다. | 관리자 UI kit, 공통 UI 컴포넌트, 목록 필터, 검색, 페이지네이션 helper |
| 요청 보호 | 저장 작업은 로그인, 권한, CSRF 검증을 통과해야 하므로 링크 클릭이나 외부 폼 제출만으로 상태가 바뀌지 않는다. | 관리자 POST action CSRF 검증, 로그인 검사 |

| 기록 기반 확인 | 관리자 작업과 회원 인증 이력을 남겨 문제가 생겼을 때 “무엇이 언제 일어났는지”를 확인할 수 있다. | 관리자 감사 로그, 회원 인증 로그 |
